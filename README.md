# Daily Meditations Email

A random passage from Marcus Aurelius's *Meditations* delivered to your inbox every morning. Runs free on GitHub Actions + Resend.

**Cost: $0/month.** Resend free tier = 3,000 emails/month (you send ~31). GitHub Actions = unlimited minutes for public repos.

---

## Setup (one-time, ~20 minutes)

### 1. Create a Resend account

1. Go to [resend.com](https://resend.com) and sign up (no credit card needed)
2. **Add your domain:** Resend → Domains → Add Domain
   - Enter any domain you own (e.g. `yourdomain.com`)
   - Resend gives you 2 DNS records (TXT + CNAME) to add at your domain registrar
   - Add them, wait a few minutes, click **Verify** in Resend
   - *No domain?* You can test with `onboarding@resend.dev` as the sender, but it won't work long-term
3. **Copy your API key:** Resend → API Keys → Create API Key

### 2. Create the GitHub repo

1. Create a **new public repository** on GitHub (e.g. `daily-meditations`)
2. Go to **Settings → Secrets and variables → Actions → New repository secret** and add these three:

   | Secret name       | Value                                         |
   |-------------------|-----------------------------------------------|
   | `RESEND_API_KEY`  | Your Resend API key                           |
   | `SENDER_EMAIL`    | Your verified sender (e.g. `stoic@yourdomain.com`) |
   | `RECIPIENT_EMAIL` | Your personal email where you want to receive passages |

### 3. Generate the text data

Run this once on your local machine to download and parse the full Meditations text:

```bash
pip install requests beautifulsoup4
python fetch_meditations.py
```

This creates `meditations.json` (~400+ passages from the George Long translation, 1862, public domain).

### 4. Push everything

```bash
git add .
git commit -m "Initial setup"
git push origin main
```

### 5. Test it

1. Go to your repo → **Actions** tab
2. Click **Daily Meditations Email** → **Run workflow** → **Run workflow**
3. Check your inbox

If the run succeeds (green check), you're done. Emails arrive daily around **7:37 AM WAT** (6:37 AM UTC). Edit the cron time in `.github/workflows/daily-email.yml` if you want a different delivery time.

---

## Changing the delivery time

Edit the `cron` line in `.github/workflows/daily-email.yml`:

```yaml
- cron: '37 6 * * *'   # minute hour * * * (UTC)
```

Cron format: `minute hour day-of-month month day-of-week`. All times are **UTC** (WAT = UTC + 1).

Some examples:
- `'0 7 * * *'` → 8:00 AM WAT
- `'0 20 * * *'` → 9:00 PM WAT
- `'0 7 * * 1-5'` → 8:00 AM WAT, weekdays only

Note: GitHub Actions cron can be delayed 5–40 minutes from the scheduled time. This is normal.

---

## How it works

- GitHub Actions runs `send_daily_email.py` on a daily cron schedule
- The script picks a random passage from `meditations.json`
- It formats a clean HTML email and sends it via Resend's API
- A keepalive action prevents GitHub from disabling the workflow after 60 days of inactivity
